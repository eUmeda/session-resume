# Security Policy

This is a personal, best-effort open-source tool provided **as-is** (see LICENSE and the
README disclaimer). There is no warranty and no guaranteed response time.

## Reporting a vulnerability
- Preferred: email **umedaeisaku@gmail.com** with the subject prefix **[SECURITY]**.
- If GitHub's **"Report a vulnerability"** button is visible on this repo
  (repo → **Security**), that private channel may also be used.
- For non-sensitive issues, a normal GitHub issue is fine.
- **Do not** include secrets, API keys, or personal data in public issues.

Fixes are best-effort and not guaranteed.
