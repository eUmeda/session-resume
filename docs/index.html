Claude Code のセッションを、止めたところから滑らかに再開する Skill。

A Claude Code skill to resume your session, smoothly, right where you left off.

このスキルは、次のセッションで 「SESSION-RESUME.md を読んで再開して」と言うだけで済むように作りました。そう使ってください。 > SESSION-RESUME.md を読んで再開して
This skill exists so that in your next session you can just say "read SESSION-RESUME.md and resume." Use it that way. > read SESSION-RESUME.md and resume

何をするもの?

セッションの末尾で起動すると、作業コンテキスト・完了/未完了タスク・ファイル構成・設計判断・次の一手を、 LLM が読みやすい構造化プロンプトとして対象ディレクトリの SESSION-RESUME.md に書き出します。

次のセッションはそれを読むだけで、文脈の再説明なしに続きから入れます。 未完了タスクは [actionable] / [deferred] / [blocked] に分類され、次にやるべき一手を理由付きで提案します。

What does it do?

Run it at the end of a session and it writes your work context, done / not-done tasks, file layout, design decisions, and the next move into SESSION-RESUME.md in the target directory — as a structured prompt an LLM reads easily.

Your next session just reads that file and continues, with no re-explaining. Open tasks are classified into [actionable] / [deferred] / [blocked], with a reasoned recommendation for what to do next.

1インストール

git clone https://github.com/eUmeda/session-resume.git
mkdir -p ~/.claude/skills
ln -s "$PWD/session-resume" ~/.claude/skills/session-resume

シンボリックリンクが使えない環境（developer mode 無効の Windows 等）は、 cp -R session-resume ~/.claude/skills/session-resume でディレクトリごとコピーでも動きます。

1Install

git clone https://github.com/eUmeda/session-resume.git
mkdir -p ~/.claude/skills
ln -s "$PWD/session-resume" ~/.claude/skills/session-resume

No symlinks (e.g. Windows without developer mode)? Copy the directory instead: cp -R session-resume ~/.claude/skills/session-resume.

2使い方は3ステップ

  • 作業を終える/中断する前に、Claude Code で /session-resume と打つ （別プロジェクトを指すなら /session-resume --dir /path/to/project）。
  • SESSION-RESUME.md が対象ディレクトリに書き出される（チャットにも要点サマリーが出る）。
  • 次のセッションで 「SESSION-RESUME.md を読んで再開して」と言う。それだけ。

--dir で対象を明示するので、プロジェクトを移動/改名した直後でも正しく動きます（cwd 非依存）。 Claude が勝手に起動することはなく、あなたが明示的に頼んだときだけ動きます。

2Three steps

  • Before you wrap up or pause, type /session-resume in Claude Code (point it elsewhere with /session-resume --dir /path/to/project).
  • SESSION-RESUME.md is written into the target directory (with a short summary in chat).
  • Next session, say "read SESSION-RESUME.md and resume." That's it.

--dir makes the target explicit, so it works even right after you move/rename a project (no cwd dependency). Claude never auto-runs it — only when you explicitly ask.

3実際の使い方

作者が 2026 年春から日々の Claude Code セッションで使ってきた、定着したパターンです。

  • 夜に閉じる → 翌朝そのまま再開（最頻） — 末尾で /session-resume、翌朝は 「読んで再開して」だけ。別セッションが前夜の状態を即復元して継続できる。
  • 長時間ジョブ/バッチの結果待ちで中断 — 投下して結果待ちの間に、次に見る output と次ステップを確定。
  • 複数プロジェクトを並行 — 末尾に各 dir で --dir 指定して実行し、続きを独立に保存。
  • 複数日・複数再起動を跨ぐ長丁場 — ストレージ整理や大規模 migration で随時更新して途切れず継続。
  • 方針転換・意思決定の記録 — 巻き戻したときの到達点と判断理由を、次の自分と Claude へ引き継ぐ。
  • コンテキスト圧縮対策 / token 節約 — 「検証は今セッション、重い解釈は次セッション」と分担を宣言。

3Real-world use cases

Patterns the author has actually used in daily Claude Code sessions since spring 2026.

  • Close at night → resume next morning (most common) — /session-resume at the end; next morning just "read it and resume." A fresh session restores last night's state and continues.
  • Pause while a long job / batch runs — lock in which output to check next and the next step.
  • Juggling multiple projects — run per --dir at session end, saving each thread independently.
  • Multi-day work across restarts — keep it updated through storage cleanups / large migrations.
  • Recording pivots and decisions — capture where you ended up and why, for your future self and Claude.
  • Context-compaction insurance / token savings — split "verify now, interpret next session."

Roadmap（将来構想・未実装）

現状は手動運用で完結します。以下はアイデア段階で、まだ実装されていません。

  • コンテキスト圧迫時の半自動代行 — 任意で Gemini / Groq などの API キーを設定しておくと、 Claude Code 本体のコンテキストが圧迫された状況で terminal を全選択して貼り付け → 外部 LLM が再開プロンプト生成を半自動で代行する案。 opt-in（キー未設定なら従来どおり手動）で検討中。

Roadmap (ideas / not yet implemented)

Today it works fully via manual operation. The items below are ideas only — not implemented yet.

  • Semi-automatic handoff under context pressure — optionally set an API key (e.g. Gemini / Groq); when Claude Code's own context is tight, select-all in the terminal and paste it, and an external LLM drafts the resume prompt. Considered as opt-in (stays fully manual with no key).
